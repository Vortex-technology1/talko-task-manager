## 📝 ДОДАТКОВІ ПЕРЕВІРКИ

### Перевірка _tg() функцій

Функції типу `_tg('ua текст', 'ru текст')` в JS модулях - потребують окремої перевірки.

**Файли для перевірки:**
- `js/modules/104-sales.js`: `function _tg(ua,ru){return window.currentLang==='ru'?ru:ua;}`
- `js/modules/100-booking.js`: `var _tg = function(ua,ru){return window.currentLang==='ru'?ru:ua;}`
- `js/modules/100b-booking-bridge.js`
- `fix_how_panel.py`, `fix_how_panel2.py` (Python скрипти з прикладами)

**Статус:** Потребує окремого аналізу якщо виникають проблеми при перемиканні мови.

### Захардкоджені тексти

Захардкоджені тексти в HTML/JS (не через window.t()) - можуть показуватись завжди українською незалежно від обраної мови.

**Статус:** Потребує перевірки якщо користувачі скаржаться на українські тексти при виборі російської мови.

## 🔧 РЕКОМЕНДАЦІЇ

1. **Якщо користувачі скаржаться на українські тексти при виборі RU:**
   - Перевірити захардкоджені тексти в HTML
   - Перевірити _tg() функції в JS модулях
   - Перевірити чи правильно зберігається вибір мови в localStorage

2. **Для покращення аналізатора:**
   - Виключити польські ключі з підрахунку відсутніх RU
   - Додати перевірку багаторядкових значень
   - Враховувати інтернаціоналізми при порівнянні UA/RU

3. **Моніторинг:**
   - Після кожного додавання нових UA ключів - додавати відповідні RU переклади
   - Використовувати `missing-ru-translations.json` як базу для нових перекладів
